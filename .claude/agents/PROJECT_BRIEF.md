# 연구행정 RAG 챗봇 — 에이전트 공용 브리프

> 모든 역할 에이전트가 작업 시작 전에 반드시 읽는 1페이지 컨텍스트.

## 한 줄 정의
중소기업 R&D 연구행정 담당자가 업로드한 PDF/HWP/법령을 근거로, **사실만** 답하는 로컬 RAG 챗봇.

## 절대 원칙 (전 역할 공통)
1. **근거 없으면 답하지 않는다** — 검색된 청크 외 추정 금지.
2. **반드시 출처 표시** — 문서명 + 조문번호/페이지 + 필요 시 원문 인용.
3. **분석이 아닌 사실 진술** — "내 생각엔..." 금지. 법/지침/사례를 그대로 인용해 결론.
4. **불확실 시 유보** — `verdict: 판단불가`로 명시.
5. **이미지 콘텐츠는 누락 가능** — 텍스트(OCR 포함)에 한정.

## 외부 통합
- **법제처 MCP**: <https://github.com/chrisryugj/korean-law-mcp> — 국가법령정보 검색용 MCP 서버.
- **HWP MCP (선택)**: <https://github.com/treesoop/hwp-mcp> — 한글 파일 파싱용. 1차 릴리스에서는 보류 (PDF 변환으로 우회).
- **Anthropic API**: Claude Code OAuth 토큰 사용 (정적 토큰 또는 Agent SDK).

## 타겟 사용자
- 중소기업 R&D 담당자 (연구비 집행 판단)
- 연구책임자 (비목별 집행 가능 여부)
- 사업기획팀 선임 (혁신법·운영요령 조문 대응)

## 대표 질문 유형
- "학생인건비로 노트북 구매 가능한가요?" → **법/지침에서 가능 여부 + 근거 조문 인용**
- "간접비 비율 초과 시 처리?" → **혁신법 시행령 + 운영요령 절차 인용**
- "제13조 제2항 전문 보여주세요" → **원문 그대로 출력**
- "별표 2 직접비 기준?" → **표 원문 그대로 출력**

## GSD 스킬 사용 규칙
- 각 역할은 자기 영역의 GSD 스킬만 호출 (역할 정의서 참조).
- 작업 전 `gsd:progress`로 현 단계 확인 권장.
- 신규 단계 추가 시 PM 에이전트만 `gsd:add-phase` 또는 `gsd:plant-seed` 호출.
- 실행은 백엔드/프론트가 `gsd:execute-phase` 또는 직접 코드 작성.
