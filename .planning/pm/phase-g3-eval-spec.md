## 2. 평가셋 카테고리 (총 ~30 케이스)

각 케이스는 다음 필드를 가진다:

```python
{
    "category": str,                      # 7개 중 하나
    "name": str,                          # 사람이 읽을 수 있는 케이스 ID
    "question": str,                      # 사용자 질문 그대로
    "expected_doc_or_article": str,       # 기대 문서/조문 ('국가연구개발혁신법 시행령 별표2', '제15조' 등)
    "should_match_keywords": list[str],   # top-K 청크에 등장해야 할 키워드 (OR or AND 명시)
    "match_mode": "any" | "all",          # 키워드 매칭 mode
    "min_recall_at_k": int,               # K (보통 3 또는 5)
    "check": Callable[[dict], bool],      # 기존 헬퍼 조합으로 PASS 람다
}
```

## 5. 백엔드 작업 시 주의

- 케이스 중 코퍼스 실측 후 조정 필요한 항목(페이지 번호, 조문 본문 키워드)은 backend 가 사전에 인덱스 scroll 로 확인 후 픽.
- `should_match_keywords` 는 가능한 한 *원문 그대로* 의 단어 사용 (의역 금지).
- check 람다는 기존 `eval_retrieval.py` 헬퍼 재사용.
- 케이스 추가/삭제 시 PM 에게 사전 보고.
